# Technical Playbook (技术面操作手册)

基于恨铁不成小猫猫的五日分类体系与技术执行纪律。

## 五类市场日分类（Five Market Day Classification）

### 1. 趋势日（Trend Day）
**特征**：
- 开盘后价格单向运动，不回测开盘区间
- 成交量放大，波动率（ATR）扩张
- 均线多头/空头排列，无反向穿越

**操作**：
- ✅ 顺势而为，持仓不动或加仓
- ✅ 止损放宽至 2×ATR（给趋势喘息空间）
- ❌ 不逆势抄底/摸顶
- ❌ 不频繁查看盘子（避免情绪干扰）

**示例**：
> NVDA 财报超预期，开盘跳空 +5%，全日在高位震荡不回补缺口 → 趋势日，持有即可。

---

### 2. 区间日（Range Day）
**特征**：
- 价格在开盘区间 60% 范围内震荡
- 成交量萎缩，波动率（ATR）收敛
- 均线纠缠，无明确方向

**操作**：
- ✅ 低买高卖，反复做 T+0（A 股不适用）
- ✅ 止损收紧至 1×ATR
- ✅ 在支撑位买入，阻力位卖出
- ❌ 不追涨杀跌（容易买在高点、卖在低点）

**示例**：
> TSLA 在 $180-$185 之间震荡三天，成交量递减 → 区间日，等突破再追。

---

### 3. 事件日（Event Day）
**特征**：
- 财报、FOMC、CPI、政策公告等催化剂落地
- 开盘跳空，波动率（ATR）急剧扩张
- 成交量暴增，买卖盘激烈博弈

**操作**：
- ✅ 等开盘 30 分钟后再进场（避免被假突破套住）
- ✅ 用期权对冲（如有），或减仓观望
- ✅ 设置紧密止损（1×ATR）
- ❌ 不赌财报方向（即使你研究很深）
- ❌ 不重仓过夜（事件后可能反转）

**示例**：
> AAPL 财报盘后发布，次日开盘跳空 +8%，但 1 小时后回落 → 事件日，等稳定后再决策。

---

### 4. 破线日（Breakdown Day）
**特征**：
- 关键支撑位/阻力位被放量突破
- 均线死叉/金叉，趋势可能发生反转
- 成交量放大，波动率（ATR）扩张

**操作**：
- ✅ 跟随突破方向，轻仓试探
- ✅ 止损设在突破位另一侧（如突破买入，止损设在突破位下方）
- ✅ 若突破失败（假突破），立即止损
- ❌ 不逆势抄底/摸顶
- ❌ 不重仓赌反转（趋势一旦形成不会轻易改变）

**示例**：
> BTC 跌破 $60,000 支撑位，成交量放大 → 破线日，轻仓做空或减仓观望。

---

### 5. 转换日（Transition Day）
**特征**：
- 趋势日/区间日/事件日/破线日之间的过渡状态
- 成交量萎缩，波动率（ATR）收窄
- 方向不明，市场在等待新的催化剂

**操作**：
- ✅ 空仓观望，等待明确信号
- ✅ 复盘前期走势，准备候选标的清单
- ✅ 检查持仓止损是否到位
- ❌ 不频繁交易（容易亏手续费和滑点）
- ❌ 不重仓赌方向

**示例**：
> NVDA 在财报前三天，价格在 $120-$125 窄幅震荡，成交量递减 → 转换日，等财报落地后再决策。

## ATR 计算与用法

### ATR（Average True Range）定义
**真实波幅（True Range, TR）**：
```
TR = max(
  (当日最高 - 当日最低),
  abs(当日最高 - 前日收盘),
  abs(当日最低 - 前日收盘)
)
```

**ATR**：TR 的 N 日指数移动平均（通常 N=14）。

### ATR 用法
| 场景 | ATR 倍数 | 说明 |
|------|----------|------|
| 正常止损 | 1.5×ATR | 给正常波动留空间 |
| 紧密止损 | 1.0×ATR | 区间日/事件日使用 |
| 宽松止损 | 2.0×ATR | 趋势日使用 |
| 目标位 | 3.0×ATR | 趋势日的目标涨幅/跌幅 |

### Python 计算 ATR
```python
import pandas as pd

def calculate_atr(df, period=14):
    """计算 ATR
    df: DataFrame with columns [high, low, close]
    period: ATR 周期（默认 14）
    """
    df = df.copy()
    df['prev_close'] = df['close'].shift(1)
    df['tr1'] = df['high'] - df['low']
    df['tr2'] = abs(df['high'] - df['prev_close'])
    df['tr3'] = abs(df['low'] - df['prev_close'])
    df['tr'] = df[['tr1', 'tr2', 'tr3']].max(axis=1)
    df['atr'] = df['tr'].rolling(window=period).mean()
    return df['atr'].iloc[-1]
```

## 仓位管理（Position Sizing）

### 固定比例法（Fixed Fractional）
**公式**：
```
仓位数量 = (账户总市值 × 单笔风险比例) / (入场价 - 止损价)
```

**示例**：
> 账户 $100,000，单笔风险 2%，$NVDA 入场价 $120，止损价 $114
> 仓位数量 = ($100,000 × 2%) / ($120 - $114) = $2,000 / $6 = 333 股
> 实际风险 = 333 × $6 = $1,998 ≈ $2,000（符合 2% 要求）

### 基于 ATR 的仓位管理
**公式**：
```
仓位数量 = (账户总市值 × 单笔风险比例) / (1.5 × ATR)
```

**示例**：
> 账户 $100,000，单笔风险 2%，$NVDA 当前价 $120，14 日 ATR = $8
> 仓位数量 = ($100,000 × 2%) / (1.5 × $8) = $2,000 / $12 = 166 股
> 止损价 = $120 - 1.5 × $8 = $108
> 实际风险 = 166 × ($120 - $108) = $1,992 ≈ $2,000（符合 2% 要求）

## 止损与止盈（Stop-Loss & Take-Profit）

### 止损设置原则
1. **技术止损**：止损设在关键支撑位下方 1-2%（避免被假突破打掉）
2. **ATR 止损**：止损 = 入场价 - 1.5×ATR（趋势日可放宽至 2×ATR）
3. **时间止损**：若持仓 N 天（如 5 天）仍未盈利，平仓离场（避免资金占用成本）
4. **事件止损**：财报/政策公告前减仓（避免黑天鹅）

### 止盈设置原则
1. **技术止盈**：止盈设在关键阻力位下方 1-2%（避免未成交就回落）
2. **ATR 止盈**：止盈 = 入场价 + 3×ATR（趋势日可使用）
3. **分批止盈**：涨 20% 卖 1/3，涨 50% 再卖 1/3，剩余 1/3 长期持有
4. **移动止损**：随着价格上涨，不断上移止损位（保护利润）

## 量价确认（Volume-Price Confirmation）

### 有效突破的特征
✅ **价格突破 + 成交量放大** → 有效突破，可追
❌ **价格突破 + 成交量萎缩** → 假突破，不追

### 量价背离
- **价涨量缩** → 上涨动能衰竭，可能回调
- **价跌量缩** → 下跌动能衰竭，可能反弹
- **价涨量增** → 上涨动能强劲，可持有
- **价跌量增** → 下跌动能强劲，可减仓

## A 股特有问题

### 涨跌停板限制
| 板块 | 涨跌停限制 |
|------|------------|
| 主板 | ±10% |
| 创业板 | ±20% |
| 科创板 | ±20% |
| 北交所 | ±30% |

**应对策略**：
- 若个股涨停，无法买入 → 买同板块其他标的，或等次日开盘
- 若个股跌停，无法卖出 → 用期权/期货对冲（如有），或等次日开盘
- 若个股连续涨停（如 5 天），不追 → 开板当日可能暴跌（"天地板"）

### T+1 结算制度
- **当天买入，次日才能卖出** → 无法做 T+0 摊低成本
- **应对策略**：
  - 不留隔夜仓（短线），或留部分仓位（中长线）
  - 用融资融券（Margin Trading）做 T+0（需要有资质）

## 心理纪律（Mental Discipline）

### 恨铁不成小猫猫的铁律
1. **市场不奖励判断，只奖励执行** → 再好的分析，执行不到位也是亏
2. **止损是唯一的对冲** → 不要幻想"明天会涨回来"
3. **不要频繁看盘** → 容易被短期波动干扰，做出错误决策
4. **不要重仓单只股票** → 分散风险，单只股票不超过账户 20%
5. **不要借钱炒股** → 杠杆会放大亏损，甚至爆仓

### 常见情绪陷阱
| 情绪 | 表现 | 应对 |
|------|------|------|
| 贪婪 | 涨了还想涨，不止盈 | 分批止盈，移动止损 |
| 恐惧 | 跌了就怕，不止损 | 严格止损，不幻想反弹 |
| 希望 | "明天会涨回来" | 接受亏损，止损离场 |
| 后悔 | "早知道就..." | 接受已发生的事，专注下一步 |

## 检查清单（Checklist）

### 开盘前（盘前准备）
- [ ] 查看今日市场日分类（趋势/区间/事件/破线/转换）
- [ ] 检查持仓止损是否到位
- [ ] 准备候选标的清单（供应链瓶颈 + 宏观顺风 + 技术突破）
- [ ] 查看今日催化剂（财报、政策、经济数据）

### 开盘后（盘中执行）
- [ ] 前 30 分钟不交易（避免情绪化操作）
- [ ] 按市场日分类执行对应策略
- [ ] 检查量价是否确认突破
- [ ] 不频繁查看盘子（避免情绪干扰）

### 收盘后（盘后复盘）
- [ ] 记录今日操作（买入/卖出/止损原因）
- [ ] 更新候选标的清单（新的供应链瓶颈/宏观变化）
- [ ] 检查持仓止损是否需要上移（保护利润）
- [ ] 复盘今日错误（如果有），记录改进点
